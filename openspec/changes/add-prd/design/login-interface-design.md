# 登录界面 UI 设计规范

## 1. 设计概述

### 设计目标
- 简洁、现代的登录界面
- 支持邮箱验证码和密码两种登录方式
- 符合 Be.run 项目的设计系统规范
- 响应式设计，支持桌面端和移动端

### 设计原则
- 清晰的视觉层次
- 友好的错误提示
- 流畅的交互动效
- 无障碍设计支持

## 6. 交互规范

### 6.1 登录方式切换

- 点击标签页切换时，表单内容淡入淡出（200ms）
- 已输入的邮箱地址保留
- 验证码/密码输入框清空

### 6.2 获取验证码

```
初始状态: "获取验证码"
点击后: 显示 60s 倒计时
倒计时期间: 按钮禁用，显示 "60s 后重发"
倒计时结束: 恢复 "获取验证码"
```

### 6.3 表单验证

#### 实时验证
- 邮箱输入后失焦时验证格式
- 验证码输入时限制只能输入数字
- 错误时输入框边框变红，下方显示错误信息

#### 提交验证
- 点击登录按钮时验证所有必填项
- 验证通过显示 loading 状态
- 验证失败聚焦到第一个错误项

### 6.4 Loading 状态

```
按钮显示: 圆形 spinner (20px) + "登录中..."
spinner: border: 2px solid rgba(255,255,255,0.3)
         border-top-color: #FFFFFF
         animation: spin 1s linear infinite
输入框: 禁用状态
```

### 6.5 过渡动画

| 动画 | 时长 | 缓动 |
|------|------|------|
| 标签切换 | 200ms | ease-in-out |
| 输入框焦点 | 150ms | ease-out |
| 按钮悬停 | 100ms | ease-out |
| 错误提示显示 | 200ms | ease-out |
| Loading 旋转 | 1000ms | linear infinite |

## 7. 响应式设计

### 7.1 断点

| 设备 | 宽度 | 容器调整 |
|------|------|---------|
| 桌面端 | >= 1024px | 420px 居中 |
| 平板端 | 768px - 1023px | 380px 居中 |
| 移动端 | < 768px | 100% - 32px |

### 7.2 移动端适配

- Logo 尺寸缩小 20%
- 容器边距减小至 24px
- 按钮保持全宽
- 字号保持不变（已使用相对单位）

## 8. 无障碍设计

### 8.1 键盘导航

- Tab 键在表单元素间切换
- Enter 键提交表单
- 焦点状态清晰可见

### 8.2 ARIA 属性

```html
<!-- 标签页 -->
<div role="tablist">
  <button role="tab" aria-selected="true">验证码登录</button>
  <button role="tab" aria-selected="false">密码登录</button>
</div>

<!-- 错误提示 -->
<div role="alert" aria-live="polite">
  请输入有效的邮箱地址
</div>

<!-- 验证码输入 -->
<input 
  type="text"
  inputmode="numeric"
  pattern="[0-9]*"
  aria-describedby="verification-code-help"
/>
```

### 8.3 对比度

- 主文字与背景对比度 >= 4.5:1
- 按钮文字与背景对比度 >= 4.5:1
- 错误提示文字对比度 >= 4.5:1
