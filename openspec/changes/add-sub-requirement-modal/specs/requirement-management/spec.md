## ADDED Requirements

### Requirement: 子需求创建弹窗
系统 SHALL 提供子需求创建弹窗，用于在父需求上下文中创建新的子需求。

#### Scenario: 弹窗基础结构
- **GIVEN** 用户正在查看父需求详情
- **WHEN** 用户点击"创建子需求"按钮
- **THEN** 系统显示子需求创建弹窗
- **AND** 弹窗宽度为 520px
- **AND** 弹窗圆角为 16px
- **AND** 弹窗背景为白色 (#FFFFFF)
- **AND** 弹窗具有阴影效果 (blur 60px, color #1A1D2E30)
- **AND** 弹窗显示在遮罩层之上 (遮罩层颜色 #1A1D2E55)

#### Scenario: 弹窗标题栏
- **GIVEN** 子需求创建弹窗已打开
- **THEN** 弹窗显示标题栏
- **AND** 标题栏高度为 56px
- **AND** 标题栏左侧显示分支图标和"新增子需求"文本
- **AND** 标题栏右侧显示关闭按钮 (X图标)
- **AND** 标题栏底部有边框 (#1A1D2E10)

#### Scenario: 父需求信息展示
- **GIVEN** 子需求创建弹窗已打开
- **THEN** 弹窗显示父需求信息区域
- **AND** 区域背景色为 #E8944A08 (橙色透明)
- **AND** 区域边框为 #E8944A22 (橙色透明)
- **AND** 区域圆角为 10px
- **AND** 显示"父需求："标签
- **AND** 显示父需求的工作项ID和名称

#### Scenario: 表单字段展示
- **GIVEN** 子需求创建弹窗已打开
- **THEN** 弹窗显示以下表单字段：
  - 子需求名称（必填，带*标记）
  - 状态（下拉选择，默认"todo"）
  - 优先级（下拉选择，继承父需求优先级）
  - 开始时间（日期选择器）
  - 截止时间（日期选择器）
  - 可获得积分（数值输入）
  - 描述（多行文本域，高度72px）
- **AND** 所有标签使用 12px 字体，font-weight 600，颜色 #1A1D2E
- **AND** 必填标记使用红色 (#F44336)
- **AND** 输入框背景为 #F5F0F0，圆角 10px，高度 38px

#### Scenario: 底部操作按钮
- **GIVEN** 子需求创建弹窗已打开
- **THEN** 弹窗底部显示操作按钮区域
- **AND** 区域高度为 60px
- **AND** 区域顶部有边框 (#1A1D2E10)
- **AND** 右侧显示"取消"按钮（灰色背景 #F5F0F0）
- **AND" 右侧显示"创建子需求"按钮（橙色背景 #E8944A，白色文字）
- **AND" 创建按钮带有加号图标

#### Scenario: 关闭弹窗
- **GIVEN** 子需求创建弹窗已打开
- **WHEN** 用户点击关闭按钮
- **THEN** 系统关闭弹窗
- **WHEN** 用户点击遮罩层
- **THEN** 系统关闭弹窗
- **WHEN** 用户按 ESC 键
- **THEN** 系统关闭弹窗
- **AND** 关闭时如有未保存数据，显示确认对话框

### Requirement: 子需求表单验证
系统 SHALL 对子需求创建表单进行验证，确保数据有效性。

#### Scenario: 名称字段验证
- **GIVEN** 用户填写子需求名称
- **WHEN** 名称为空
- **THEN** 系统显示错误提示"子需求名称不能为空"
- **WHEN** 名称少于2个字符
- **THEN** 系统显示错误提示"子需求名称至少需要2个字符"
- **WHEN** 名称超过100个字符
- **THEN** 系统显示错误提示"子需求名称不能超过100个字符"
- **AND** 验证在字段失焦时触发

#### Scenario: 时间字段验证
- **GIVEN** 用户选择开始时间和截止时间
- **WHEN** 截止时间早于开始时间
- **THEN** 系统显示错误提示"截止时间不能早于开始时间"
- **AND** 禁用提交按钮直到错误修正

#### Scenario: 积分字段验证
- **GIVEN** 用户填写可获得积分
- **WHEN** 积分为负数
- **THEN** 系统显示错误提示"积分不能为负数"
- **AND" 积分为非数字时显示"请输入有效的数字"

#### Scenario: 表单提交前验证
- **GIVEN** 用户点击"创建子需求"按钮
- **WHEN** 表单存在验证错误
- **THEN** 系统不提交表单
- **AND** 在第一个错误字段处聚焦
- **AND** 显示所有错误提示

### Requirement: 子需求创建API
系统 SHALL 提供API接口用于创建子需求，并正确处理父子关系。

#### Scenario: 创建子需求成功
- **GIVEN** 用户提交有效的子需求数据
- **WHEN** 调用 `POST /api/requirements`
- **AND** 请求体包含 `parentId` 字段
- **THEN** 系统创建新的子需求
- **AND** 自动生成子需求工作项ID（格式：父ID-序号，如 REQ-001-1）
- **AND" 子需求的 `parent_id` 字段设置为父需求ID
- **AND" 子需求的 `project_id` 与父需求相同
- **AND" 系统返回 `{ success: true, requirement: Requirement }`
- **AND" HTTP 状态码为 201

#### Scenario: 验证父需求存在
- **GIVEN** 用户提交子需求数据
- **WHEN** 指定的父需求不存在
- **THEN** 系统返回 `{ success: false, message: '父需求不存在' }`
- **AND** HTTP 状态码为 404

#### Scenario: 验证项目权限
- **GIVEN** 用户提交子需求数据
- **WHEN** 父需求属于用户无权限访问的项目
- **THEN** 系统返回 `{ success: false, message: '无权限在此项目创建需求' }`
- **AND** HTTP 状态码为 403

#### Scenario: 防止循环引用
- **GIVEN** 用户提交子需求数据
- **WHEN** 尝试将需求设为自己的子需求
- **THEN** 系统返回 `{ success: false, message: '不能将需求设为自己的子需求' }`
- **AND** HTTP 状态码为 400

#### Scenario: 工作项ID唯一性
- **GIVEN** 系统生成子需求工作项ID
- **WHEN** 生成的ID已存在
- **THEN** 系统递增序号重新生成
- **AND** 直到找到唯一的ID

### Requirement: 操作日志记录
系统 SHALL 记录子需求创建操作，便于后续审计。

#### Scenario: 记录创建操作
- **GIVEN** 子需求创建成功
- **THEN** 系统在 operation_logs 表中创建记录
- **AND** `entity_type` 为 'requirement'
- **AND** `entity_id` 为新创建子需求的ID
- **AND** `action` 为 'create'
- **AND** `user_id` 为当前操作用户ID
- **AND** `created_at` 为当前时间

#### Scenario: 记录父子关系
- **GIVEN** 子需求创建成功且指定了父需求
- **THEN** 操作日志记录包含父需求信息
- **AND** 可在操作历史中查看到父子关系建立记录

## MODIFIED Requirements

### Requirement: 子需求列表组件
系统 SHALL 在子需求列表中提供创建子需求的入口。

#### Scenario: 显示创建按钮
- **GIVEN** 用户查看需求详情面板的子需求区域
- **THEN** 子需求列表头部显示"创建子需求"按钮
- **AND** 按钮样式为文字链接（橙色 #E8944A）
- **AND** 按钮带有加号图标
- **AND" 当子需求为空时，空状态也显示"创建一个"链接

#### Scenario: 打开创建弹窗
- **GIVEN** 用户点击"创建子需求"按钮
- **THEN** 系统打开子需求创建弹窗
- **AND" 弹窗自动填充父需求信息
- **AND** 表单字段使用默认值（优先级继承父需求）

#### Scenario: 刷新列表
- **GIVEN** 用户成功创建子需求
- **WHEN** 弹窗关闭
- **THEN** 子需求列表自动刷新
- **AND" 新创建的子需求显示在列表中
- **AND** 显示成功提示"子需求创建成功"
