# Change: 集成 Vitest + Playwright 自动化测试体系

## 为什么
当前项目仅有 1 个示例 E2E 测试，缺乏单元测试和完整的测试体系。代码变更后无法自动验证正确性，导致：
- 回归错误难以发现
- 重构信心不足
- 代码质量无法保障

## 什么变更
- **引入 Vitest**：配置单元测试框架，覆盖组件、工具函数、类型验证
- **扩展 Playwright**：补充 API 集成测试，覆盖认证、数据 CRUD 流程
- **修改 OpenSpec 流程**：所有变更实施后必须通过测试验证
- **创建测试规范**：定义测试覆盖标准和验收准则

## 影响
- 新增规范：testing
- 新增配置文件：`vitest.config.ts`、`tests/vitest-setup.ts`
- 新增测试目录：`tests/unit/`、`tests/integration/`
- 变更实施流程：代码实现 → 编写测试 → 运行测试 → 验证通过
- 当前 18 个活跃变更需要逐步补齐测试覆盖
