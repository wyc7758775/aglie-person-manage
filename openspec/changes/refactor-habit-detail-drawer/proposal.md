# 变更：重构习惯详情抽屉界面

## 背景

当前的习惯详情抽屉 (`TaskDetailDrawer.tsx`) 与设计稿存在显著差异，需要进行全面的UI重构以确保严格遵循设计规范。

## 现状问题

当前实现与设计稿 (Node ID: OD72U) 的主要差异：

1. **头部区域**
   - 当前：显示"任务详情"标题和编辑按钮
   - 设计稿：显示任务ID（可复制的标签）、关闭按钮

2. **标题行**
   - 当前：只有任务标题和"标记完成"按钮
   - 设计稿：任务标题 + 完成计数器（完成 7/12，带+/-按钮）

3. **统计卡片**
   - 当前：2x2网格（连胜天数、经验值、金币奖励、完成次数）
   - 设计稿：3列布局（连续天数、金币、生命值）

4. **信息网格**
   - 当前：显示难度、优先级、开始日期、截止日期、重复日、标签
   - 设计稿：显示难度、频率、金币奖励（+3.5 G/次），附带编辑规则提示

5. **热力图**
   - 当前：无
   - 设计稿：完整的热力图组件（KPI指标、月份标签、网格、图例）

6. **趋势图**
   - 当前：无
   - 设计稿：周完成趋势线

7. **备注区域**
   - 当前：只读文本区域
   - 设计稿：可直接编辑的输入框

## 变更内容

### UI 实现标准

**重要：本变更要求像素级还原设计稿，必须通过 pencil mcp 工具调用设计稿数据进行精确实现。**

- 所有颜色值必须直接从设计稿节点读取（使用 pencil mcp 获取 fillColor, textColor 等属性）
- 所有尺寸（width, height, padding, margin, gap）必须匹配设计稿精确数值
- 所有字体（fontFamily, fontSize, fontWeight）必须与 design.md 中规范一致
- 所有圆角（cornerRadius）和阴影（shadow）效果必须严格按设计稿实现
- 组件布局结构必须与设计稿层次结构完全一致

### UI 结构重构

将 `TaskDetailDrawer.tsx` 重构为严格遵循设计稿的布局（Node ID: OD72U）：

- **PanelHeader**: 类型标签 + 任务ID（可复制）+ 关闭按钮
- **titleRow**: 任务标题 + 完成计数器
- **statsCard**: 深色卡片，3列统计（连续天数、金币、生命值）
- **infoGrid**: 浅色卡片，3列信息（难度、频率、金币奖励）
- **descSection**: 可编辑的备注区域
- **heatmapSection**: 习惯热力图组件
- **trendSection**: 周完成趋势线

### 新增组件

1. **HabitHeatmap** - 习惯热力图组件
2. **HabitTrendChart** - 周趋势图表组件
3. **CompletionCounter** - 完成计数器（+/-按钮）

### 交互优化

- 完成任务次数的快速增减（+/-按钮）
- 备注的直接编辑（使用富文本编辑器，无需切换编辑模式）
- 难度、频率、金币奖励三个配置的 hover 编辑
- 关闭抽屉时的未保存更改确认
- 任务ID的一键复制
