# Proposal: update-login-interface

## Why

当前登录界面仅支持用户名+密码登录方式，存在以下问题：
1. **用户体验不佳**：用户需要记忆密码，新用户注册流程独立于登录，增加操作步骤
2. **安全性局限**：仅依赖密码认证，缺少验证码等辅助验证手段
3. **新用户门槛高**：需要先注册再登录，转化率低

PRD 来源：`product-designs/login-interface-20260301/prd.md`

## What Changes

### 1. 登录方式升级
- **新增**邮箱验证码登录作为主要登录方式（P0）
- **保留**密码登录作为备选方式，改为邮箱+密码
- **新增**登录方式切换标签（验证码登录 | 密码登录）
- **移除**用户名登录字段，统一使用邮箱标识用户

### 2. 自动注册机制
- 新用户通过邮箱验证码首次登录时，系统自动创建账户
- 昵称默认为邮箱前缀
- 无需设置初始密码

### 3. 验证码服务
- 新增验证码发送/验证 API
- 60 秒发送间隔，5 分钟有效期
- 24 小时内同一邮箱最多 10 次（防刷）

### 4. 安全增强
- 连续 5 次密码错误锁定账户 15 分钟
- 邮箱/密码错误统一返回模糊提示（防枚举）

### 5. 界面重构
- 品牌展示（Be.run Logo + 欢迎回来）
- 表单校验（邮箱实时校验、验证码6位数字限制、密码6-32字符）
- 密码显示/隐藏切换
- 加载状态与错误提示

### 6. i18n 支持
- 新增 `auth.*` 命名空间的翻译键
- 支持中文、English、日本語三语

## Impact

### 受影响的规格
- **auth** — 登录流程、登录界面默认填充、Tab导航、回车提交等需求均需修改

### 受影响的文件（预估）
- `apps/web/app/page.tsx` — 登录页面重构
- `apps/web/app/lib/auth.ts` — 认证逻辑扩展
- `apps/web/app/lib/auth-db.ts` — 数据库操作（邮箱查询、验证码存储）
- `apps/web/app/lib/db.ts` — 新增验证码表
- `apps/web/app/lib/definitions.ts` — 新增类型定义
- `apps/web/app/api/auth/` — 新增验证码发送/验证 API
- `apps/web/app/lib/i18n/` — 新增翻译字典

### 不受影响
- 路由保护逻辑（middleware.ts）— 保持不变
- 退出登录功能 — 保持不变
- 用户角色系统 — 保持不变
