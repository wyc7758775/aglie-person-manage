## 1. 背景与目标

### 1.1 背景

Be.run（Agile Person Manage）是一个个人敏捷项目管理系统，用户需要通过登录验证身份后才能访问个人数据。当前系统采用自定义认证实现（非 NextAuth），需要设计一套简洁、安全、用户体验良好的登录界面。

### 1.2 目标

- 提供流畅的登录体验，支持邮箱验证码和密码两种登录方式
- 新用户首次登录自动完成注册，降低使用门槛
- 登录后智能跳转，提升用户操作效率
- 确保基础安全（密码加密 + HTTPS），保护用户账户

### 1.3 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|----------|
| 登录成功率 | ≥ 95% | 登录成功次数 / 尝试登录次数 |
| 平均登录耗时 | ≤ 30 秒 | 进入登录页到登录成功的时间 |
| 新用户注册转化率 | ≥ 90% | 首次登录完成数 / 首次登录尝试数 |

## 2. 用户与场景

### 2.1 目标用户

| 用户类型 | 描述 | 核心诉求 |
|----------|------|----------|
| 普通用户 | 使用系统进行个人项目管理的用户 | 快速登录，无需记住复杂密码 |
| 回归用户 | 已注册并使用过系统的用户 | 安全便捷地访问已有数据 |

### 2.2 使用场景

| 场景 | 描述 | 频率 |
|------|------|------|
| 日常登录 | 用户打开应用，使用邮箱验证码登录 | 高 |
| 密码登录 | 用户选择密码登录方式，输入邮箱和密码 | 中 |
| 新用户首次登录 | 新用户通过邮箱验证码自动完成注册 | 中 |
| Session 过期重登 | 用户 session 过期后被拦截到登录页 | 低 |

### 2.3 用户故事

- **作为** 普通用户，**我希望** 使用邮箱验证码快速登录，**以便** 无需记忆密码即可访问系统
- **作为** 回归用户，**我希望** 使用密码登录，**以便** 在没有邮箱访问权限时也能登录
- **作为** 新用户，**我希望** 首次登录时自动完成注册，**以便** 快速开始使用系统
- **作为** 被拦截用户，**我希望** 登录后回到之前访问的页面，**以便** 继续未完成的工作

## 3. 功能需求

### 3.1 功能列表

| 功能模块 | 功能点 | 优先级 | 说明 |
|----------|--------|--------|------|
| 登录方式切换 | 支持验证码/密码登录切换 | P0 | 默认展示验证码登录 |
| 邮箱验证码登录 | 发送验证码、验证登录 | P0 | 主要登录方式 |
| 密码登录 | 邮箱 + 密码验证 | P0 | 备选登录方式 |
| 自动注册 | 新用户首次登录自动创建账户 | P0 | 无感注册体验 |
| 智能跳转 | 登录后回到原页面或首页 | P0 | 优先回到登录前页面 |
| 验证码策略 | 发送频率和有效期控制 | P1 | 60秒重发，5分钟有效 |

### 3.2 功能详细描述

#### 功能：邮箱验证码登录

**描述**：用户使用邮箱地址接收验证码，输入验证码后完成登录。

**用户流程**：
1. 用户输入邮箱地址
2. 点击「获取验证码」按钮
3. 系统发送验证码到用户邮箱
4. 用户在验证码输入框输入收到的验证码
5. 点击「登录」按钮
6. 系统验证通过后登录成功，跳转至目标页面

**业务规则**：
- 邮箱格式需符合标准邮箱格式（xxx@xxx.xxx）
- 验证码发送间隔为 60 秒，倒计时期间按钮置灰
- 验证码有效期为 5 分钟，过期需重新获取
- 同一邮箱 24 小时内最多发送 10 次验证码（防刷保护）

**异常处理**：
- 邮箱格式错误 → 输入框下方提示「请输入有效的邮箱地址」
- 验证码错误 → 提示「验证码错误，请重新输入」
- 验证码过期 → 提示「验证码已过期，请重新获取」
- 发送过于频繁 → 提示「发送过于频繁，请稍后再试」

#### 功能：密码登录

**描述**：用户使用已注册的邮箱和密码进行登录。

**用户流程**：
1. 用户点击「密码登录」切换登录方式
2. 输入邮箱地址和密码
3. 点击「登录」按钮
4. 系统验证邮箱和密码匹配后登录成功

**业务规则**：
- 密码需经过 bcrypt 加密存储
- 密码输入框默认隐藏内容，支持显示/隐藏切换
- 连续 5 次密码错误后锁定账户 15 分钟

**异常处理**：
- 邮箱不存在 → 提示「邮箱或密码错误」（模糊提示防枚举）
- 密码错误 → 提示「邮箱或密码错误」
- 账户被锁定 → 提示「账户已锁定，请 15 分钟后重试」

#### 功能：自动注册

**描述**：新用户使用邮箱验证码登录时，系统检查邮箱是否已注册，若未注册则自动创建账户。

**业务规则**：
- 自动注册用户昵称默认为邮箱前缀（如 user@example.com → user）
- 用户可在设置中修改昵称和其他信息
- 自动注册不设置初始密码，用户可在设置中设置密码

| 项目 | 内容 |
|------|------|

### 4.2 页面布局

登录页面包含以下元素：
- 系统 Logo 和名称（Be.run）
- 登录标题「欢迎回来」
- 登录方式切换标签（验证码登录 | 密码登录）
- 邮箱输入框
- 验证码/密码输入框
- 「获取验证码」按钮（仅验证码登录时显示）
- 「登录」按钮
- 版权信息

### 4.3 交互逻辑表

| 页面/组件 | 交互点 | 触发 | 前置条件 | 行为 | 结果/状态 | 异常/边界 |
|-----------|--------|------|----------|------|-----------|-----------|
| 登录页 | 页面加载 | - | - | 检查是否已登录 | 已登录→跳转首页；未登录→展示登录页 | - |
| 登录页 | 登录方式标签切换 | 点击 | - | 切换验证码/密码登录表单 | 对应表单显示，另一表单隐藏 | - |
| 验证码登录 | 邮箱输入 | 输入/粘贴 | - | 实时校验邮箱格式 | 格式错误→边框变红 | 空值不校验 |
| 验证码登录 | 邮箱失焦 | 失焦 | 有内容 | 校验邮箱格式 | 错误→下方显示提示文案 | 空值不校验 |
| 验证码登录 | 「获取验证码」按钮 | 点击 | 邮箱格式正确 | 1. 按钮置灰开始倒计时 2. 调用发送 API | 显示倒计时（60s） | 邮箱格式错误→提示；发送失败→toast 提示 |
| 验证码登录 | 验证码输入 | 输入 | - | 限制只能输入 6 位数字 | 自动过滤非数字字符 | - |
| 验证码登录 | 「登录」按钮 | 点击 | 邮箱+验证码已填 | 1. 按钮 loading 2. 调用登录 API | 成功→跳转目标页 | 校验失败→字段标红；失败→toast 提示 |
| 密码登录 | 「登录」按钮 | 点击 | 邮箱+密码已填 | 1. 按钮 loading 2. 调用登录 API | 成功→跳转目标页 | 失败→toast 提示；连续错误→锁定提示 |

### 4.4 校验规则

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| 邮箱 | 必填，符合邮箱格式（正则：/^[^\s@]+@[^\s@]+\.[^\s@]+$/) | 请输入邮箱地址 / 请输入有效的邮箱地址 |
| 验证码 | 必填，6 位数字 | 请输入验证码 / 验证码为 6 位数字 |
| 密码 | 必填，6–32 字符 | 请输入密码 / 密码长度需在 6–32 字符之间 |

### 4.5 状态说明

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|----------|----------|
| 初始 | 页面加载完成，表单为空 | 页面进入 | 用户开始输入 |
| 验证码发送中 | 正在请求发送验证码 | 点击「获取验证码」 | API 返回 |
| 验证码倒计时 | 60 秒倒计时期间 | 验证码发送成功 | 倒计时结束 |
| 登录中 | 正在请求登录验证 | 点击「登录」 | API 返回 |
| 登录成功 | 验证通过，准备跳转 | 登录 API 成功 | 页面跳转完成 |
| 登录失败 | 验证未通过 | 登录 API 失败 | 用户重新输入或重试 |

## 5. 非功能需求

| 类型 | 要求 |
|------|------|
| 性能 | 页面首屏加载时间 ≤ 2 秒；登录 API 响应时间 ≤ 200ms |
| 安全 | 密码使用 bcrypt 加密存储；所有接口使用 HTTPS；防止 CSRF 攻击；验证码防暴力破解（频率限制） |
| 兼容性 | 支持 Chrome、Firefox、Safari、Edge 最新两个版本；支持移动端浏览器 |
| 可用性 | 输入框支持自动填充；支持键盘 Tab 切换焦点；支持 Enter 键提交 |

## 7. 范围与排期

### 7.1 本期范围（In Scope）

- 邮箱验证码登录（主流程）
- 邮箱密码登录（备选流程）
- 登录方式切换
- 验证码发送策略（60秒/5分钟）
- 自动注册逻辑
- 智能跳转逻辑
- 基础安全防护

### 7.2 下期范围（Out of Scope）

- 第三方登录（微信、GitHub、Google 等）—— 需接入 OAuth，优先级较低
- 双因素认证（2FA）—— 当前安全级别为「基础」，暂不需要
- 记住密码/自动登录 —— 可通过 session 延长实现，本期不单独开发
- 密码找回功能 —— 依赖邮件服务，下期规划
